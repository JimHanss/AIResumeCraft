# 编辑器布局与评分信息精简

## 目标

精简简历编辑器工作台顶部的信息层级，移除当前页面中重复或干扰编辑的状态展示，让评分信息出现在更稳定的位置。完成后，用户进入编辑器时应直接看到左侧模块选择、中间简历编辑和右侧预览，不再被“模块数量”“本地草稿”“简历搭建”“编辑工作台”等辅助信息占用注意力；评分保留在全局 header 中，雷达评分移动到左侧模块选择下方。

## 用户

- 需要高频编辑简历内容并关注预览效果的简历创建者。
- 需要快速判断简历评分但不希望顶部区域拥挤的使用者。
- 需要维护编辑器信息架构、响应式布局和测试覆盖的开发者。

## 用户场景

- 作为简历创建者，我打开编辑器后，可以更快进入模块编辑和预览，不会看到多余的工作台标题和模块数量卡片。
- 作为简历创建者，我可以在页面顶部 header 中持续看到当前评分。
- 作为简历创建者，我可以在左侧模块选择区域下方查看雷达评分，并且它不会挤占中间编辑区或右侧预览区。
- 作为移动端或窄屏用户，我仍然可以访问评分和雷达图，页面不产生横向溢出。

## 范围内

- 移除工作区顶部的“模块数量”显示。
- 将当前评分显示移动到全局 header 中。
- 移除工作区顶部或工具区中的“本地草稿”显示。
- 将雷达评分组件移动到左侧模块选择下方。
- 移除左上角工作区标题文案“简历搭建”和“编辑工作台”。
- 保留中间编辑区标题“简历编辑”或等价主编辑区标题。
- 保留已有评分数值来源和雷达图数据来源，不改变评分算法。
- 调整桌面和移动端布局，确保 header、左侧模块区、雷达图、中间编辑区、右侧预览区不互相遮挡。
- 更新新增或移动后的可见文案对应的中英文 i18n 文案。
- 更新相关组件测试或 E2E 断言，覆盖评分位置、雷达图位置和被移除元素不可见。

## 范围外

- 修改评分算法或引入真实 AI 评分。
- 修改雷达图维度、样式主题或 ECharts 数据结构。
- 重做整体三栏布局、编辑器主题系统或 PDF 导出功能。
- 移除本地草稿持久化能力；本次只隐藏“本地草稿”可见状态。
- 改动简历模块数据结构或模块排序逻辑。

## 验收标准

- 工作区顶部不再显示“模块数量”卡片或等价模块计数信息。
- 全局 header 中可见当前评分，例如中文显示“评分 80”，英文显示等价文案。
- 当前评分在页面滚动时随 sticky header 保持可见。
- 页面中不再显示“本地草稿”状态标签或按钮。
- 页面左上工作区不再显示“简历搭建”和“编辑工作台”。
- 雷达评分组件显示在左侧模块选择区域下方。
- 雷达评分在桌面端不遮挡模块列表和“添加模块”按钮。
- 雷达评分在移动端或窄屏布局下仍可访问，且不导致 body 横向溢出。
- 中间编辑区标题保持为“简历编辑”或当前等价文案，不被误删。
- 原有主题切换、撤销/重做、PDF 导出、模块拖拽和预览同步功能不回退。
- 中文和英文 UI 下，评分相关文案不出现 raw i18n key。
- 相关测试覆盖元素移除、评分 header 展示和雷达图左侧展示。
- `corepack yarn lint`、`corepack yarn typecheck`、`corepack yarn test:unit` 和相关 E2E 测试通过。

## 边界情况

- 当评分数据缺失时，header 中应显示安全默认值或隐藏数值状态，不应展示 `NaN`、`undefined` 或 raw key。
- 当左侧模块数量较多时，雷达评分不应把模块列表挤到不可操作的位置；必要时左侧栏可滚动。
- 当页面宽度不足以维持三栏布局时，评分和雷达图应随现有响应式布局重新排列。
- 深色模式和不同简历主题下，header 评分与左侧雷达评分仍应可读。
- PDF 导出不应捕获 header 评分或左侧雷达图，只导出简历预览区域。

## 假设

- 当前评分继续使用 `store.document.score.overall` 或现有等价数据源。
- Header 指全局 `.app-header`，即包含品牌、语言选择和深色模式按钮的吸顶区域。
- 雷达评分移动到左侧模块选择下方，属于编辑器辅助信息，不进入简历预览纸张。
- “本地草稿”只是可见状态提示，隐藏它不会影响 localStorage 草稿保存和恢复。
- 本次优先做布局和信息层级调整，不新增新的图表或指标。

## 需要澄清

- 无阻塞澄清项；上述假设可直接进入技术计划。
