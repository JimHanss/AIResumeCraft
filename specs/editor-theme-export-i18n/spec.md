# 编辑器主题、导出与国际化

## 目标

将当前可用的简历编辑器从基础 MVP 提升为更完整、更精致的简历创作工具。编辑器需要支持多套简历主题切换、完整 UI 文案国际化、A4 PDF 导出、撤销/重做，以及可见的简历评分雷达图。里程碑是：编辑器既好看又好用，用户可以完成简历编辑、换肤、导出 PDF、中英切换，并且在常见桌面和移动端宽度下没有明显视觉问题。

## 用户

- 需要快速编辑、换肤并导出成品 PDF 的简历创建者。
- 需要验证编辑器是否具备完整基础工作流的产品和设计评审者。
- 需要维护编辑器体验、国际化文案、预览渲染和测试覆盖的开发者。

## 用户场景

- 作为简历创建者，我可以在多套简历主题之间切换，并立即看到右侧预览更新。
- 作为简历创建者，我可以将编辑器 UI 切换为中文或英文，界面中不出现混杂或未翻译文案。
- 作为简历创建者，我可以将当前简历预览导出为 A4 PDF，并选择导出质量。
- 作为简历创建者，我可以撤销和重做最近的编辑，包括内容修改和模块排序。
- 作为简历创建者，我可以看到基于初始模拟数据的简历评分雷达图。
- 作为开发者，我可以运行组件测试，覆盖拖拽、表单校验和导出触发行为。

## 范围内

- 提供 2 到 3 套可选的简历视觉主题。
- 切换主题时，右侧简历预览必须即时更新，不需要刷新页面。
- 主题选择应随当前本地简历或编辑器状态持久化。
- 主题之间需要有明确视觉差异，例如字号层级、强调色、章节标题样式、间距和页面底色。
- 保留现有偏浅蓝的编辑器外壳，同时允许简历预览主题与编辑器外壳风格不同。
- 本功能新增或改动的可见 UI 文案都需要抽离到现有 i18n 消息结构中。
- 支持中英文切换，覆盖工具栏、导出控件、主题名称、质量选项、撤销/重做控件和评分文案。
- 增加 PDF 导出功能，只导出简历预览区域，不包含左侧物料区、中间编辑区、工具栏或浏览器 UI。
- PDF 默认目标为 A4 纵向页面。
- PDF 导出需要至少提供两个质量选项，例如标准质量和高质量。
- 导出过程中需要有可见的加载态或禁用态，避免重复触发导出。
- 实现本地编辑的撤销和重做。
- 撤销和重做应尽量覆盖简历内容编辑、模块新增/删除、模块排序、主题切换和预览设置。
- 在编辑器或预览工作区中增加简历评分雷达图，初期使用模拟评分数据。
- 保持现有本地优先模型；主题、导出、评分和历史栈不依赖后端服务。
- 增加组件测试或集成测试，覆盖拖拽行为、表单校验行为和导出触发行为。

## 范围外

- AI 简历评分、改写、建议或流式模型输出。
- 服务端 PDF 生成。
- Word、图片或 ZIP 导出。
- 完整模板市场、付费主题或远程主题下载。
- 云端保存、账号登录或多设备同步。
- 真实 ATS 评分逻辑；本阶段只需要确定性的模拟评分数据。
- 支持所有纸张尺寸的完整打印布局。
- 超出现有 Vue i18n 体系之外的翻译管理工作流。

## 验收标准

- 编辑器提供 2 到 3 套可选简历主题。
- 切换主题后，右侧简历预览立即发生可见变化。
- 刷新页面后，已选择的主题仍能恢复。
- 主题样式不会污染编辑器外壳控件。
- 中文和英文 UI 切换继续可用。
- 主题、导出、导出质量、撤销、重做和评分雷达图相关的新 UI 文案都有中文和英文翻译。
- 简历预览可以导出为 A4 纵向 PDF。
- 用户可以在触发导出前选择导出质量。
- PDF 导出运行期间，导出控件能防止重复点击。
- 导出的 PDF 包含当前简历预览内容和当前主题样式。
- 撤销和重做控件可见且可用。
- 撤销可以恢复内容编辑和排序变更前的状态。
- 重做可以恢复被撤销的状态。
- 在撤销后执行新的编辑时，重做分支会被清空。
- 撤销和重做历史在首版中不需要跨刷新持久化。
- 简历评分雷达图使用确定性的模拟数据渲染。
- 雷达图在常见桌面和移动端宽度下响应式展示，不溢出工作区。
- 组件测试覆盖拖拽行为、必填表单校验行为和 PDF 导出触发行为。
- `yarn lint`、`yarn typecheck`、`yarn test:unit` 以及相关组件或 E2E 测试命令通过。
- 切换主题和操作导出控件后，编辑器外壳、中间画布、预览工具栏和右侧预览区没有明显视觉问题。

## 边界情况

- 简历预览为空时导出，应展示明确的本地化禁用态或空状态，而不是生成损坏 PDF。
- 图片缺失或图片 URL 无效时，导出仍应使用安全占位内容生成 PDF。
- 简历内容过长时，不应重叠或溢出页面边界；可以根据实现选择缩放、分页，或给出清晰的本地化失败提示。
- 切换主题不应意外重置简历内容、模块顺序或当前表单焦点。
- 撤销模块删除时，应恢复被删模块的内容和排序位置。
- 撤销主题切换时，不应改变简历内容。
- 快速连续编辑不应造成历史栈过大或 UI 卡死。
- 浏览器画布或 PDF API 失败时，编辑器应展示本地化错误状态并可恢复。
- 在移动端宽度下，导出和主题控件仍应可操作，且不产生横向溢出。
- 缺失翻译 key 不应以原始 key 路径形式出现在可见 UI 中。

## 假设

- 现有 Vue 3 编辑器、Pinia store、Naive UI 控件和 vue-i18n 设置继续作为基础。
- 简历预览主题可以建模为本地主题定义，并通过 CSS 变量或等价的局部样式机制应用到预览区域。
- PDF 导出为客户端导出，并按要求使用 html2canvas 与 jsPDF。
- 首版 A4 导出目标为纵向页面。
- 导出捕获右侧简历预览纸张区域，而不是整个编辑器工作区。
- 初始评分雷达图数据为模拟数据，不需要代表真实简历质量。
- 撤销和重做历史首版为当前会话内状态，不跨刷新持久化。
- 现有 localStorage 简历持久化仍是持久化简历数据的来源。

## 需要澄清

- 首轮技术计划无需额外澄清；上述未定细节可先按假设推进。
