# 自定义模块构建器

## 目标

在“添加模块”流程中引入弹窗式自定义模块编辑区，让用户可以创建不局限于固定简历模板的模块。用户应能在弹窗中设置模块名、字段标题、输入框大小、列表区域，以及字段在模块内的响应式网格分布。创建后的模块需要进入中间简历编辑区和右侧预览区，并继续支持编辑、排序、删除、持久化和主题预览。

## 用户

- 需要添加非标准简历模块的简历创建者，例如项目补充、获奖经历、证书、研究经历、作品集、校园经历等。
- 需要快速拼装不同岗位简历结构的求职者。
- 需要验证自定义模块数据结构、编辑体验、预览同步和本地持久化的产品、设计和开发人员。

## 用户场景

- 作为简历创建者，我点击“添加模块”后，可以在弹窗中创建一个新的自定义模块，而不是只能添加预设模块。
- 作为简历创建者，我可以设置模块名称，例如“证书”“项目补充”“获奖经历”。
- 作为简历创建者，我可以添加普通输入框，并为每个输入框设置字段标题、占位提示、宽度和高度。
- 作为简历创建者，我可以添加列表区，用于维护多条经历、要点、证书、链接或项目条目。
- 作为简历创建者，我可以调整字段在模块中的位置和分布，使模块内容更符合当前简历结构。
- 作为简历创建者，我保存自定义模块后，可以在中间编辑区继续编辑内容，并在右侧预览区立即看到结果。
- 作为回访用户，我刷新页面后，自定义模块结构、内容、排序和标题仍然保留。

## 范围内

- 点击“添加模块”时打开弹窗式模块构建器。
- 弹窗宽度和高度需要适合桌面编辑，不应全屏铺满；在窄屏下可响应式占满可用宽度。
- 弹窗中支持设置模块名，模块名用于中间编辑区标题和右侧预览区标题。
- 支持添加、编辑、删除和排序自定义字段。
- 自定义字段首版至少支持以下类型：
  - 单行输入框。
  - 多行输入框。
  - 列表区。
- 每个字段支持配置字段标题、占位提示、宽度、最小高度或显示高度。
- 字段位置分布支持响应式网格配置，例如整行、二分之一、三分之一、双列排列，以及字段顺序拖拽调整。
- 列表区支持新增、删除、编辑和排序列表项。
- 保存自定义模块后，模块进入当前简历模块数组，和已有模块共享排序、展开、删除、改名、撤销/重做和本地持久化能力。
- 右侧预览区需要按用户配置的字段顺序和布局展示自定义模块。
- 自定义模块在中文和英文 UI 下不出现 raw i18n key；内置控件文案需要接入现有 i18n。
- 添加必要测试覆盖自定义模块创建、字段配置、内容编辑、预览同步和刷新恢复。

## 范围外

- 完整低代码页面搭建器或任意像素级绝对定位画布。
- 复杂表格、图片上传、文件附件、视频、代码块或嵌套组件。
- 多人协作编辑、云端模板库、远程模块市场或账号同步。
- AI 自动生成自定义模块结构或内容。
- 服务端保存、数据库迁移或后端 schema 管理。
- PDF 导出分页策略的深度重做；本功能只要求自定义模块能进入现有导出内容。
- 自定义模块跨简历复用、导入导出模块模板包或模板版本管理。

## 验收标准

- 点击“添加模块”后出现自定义模块编辑弹窗。
- 弹窗不是全屏宽度；桌面端有清晰最大宽度，移动端不产生横向溢出。
- 用户不填写模块名时不能保存，并看到本地化校验提示。
- 用户可以输入模块名并保存为新的自定义模块。
- 用户可以新增单行输入框，并配置标题、占位提示、宽度和高度。
- 用户可以新增多行输入框，并配置标题、占位提示、宽度和高度。
- 用户可以新增列表区，并添加、删除、编辑、排序列表项。
- 用户可以调整字段顺序和基础宽度分布，保存后中间编辑区和右侧预览区一致。
- 自定义模块保存后显示在中间简历编辑区，并默认展开。
- 自定义模块保存后显示在右侧预览区，并跟随模块排序变化。
- 修改自定义模块内容时，右侧预览区实时同步。
- 删除自定义模块后，左侧状态、中间编辑区和右侧预览区都同步更新。
- 刷新页面后，自定义模块结构、内容、模块标题、字段顺序和模块排序仍然恢复。
- 自定义模块参与撤销/重做，不破坏现有历史栈行为。
- 中文和英文界面下，弹窗标题、字段类型、保存、取消、删除、添加字段、校验提示等文案都有对应翻译。
- 自定义模块在浅色、深色和当前简历主题下保持可读。
- 桌面和移动端都不出现明显横向溢出。
- `corepack yarn lint`、`corepack yarn typecheck`、`corepack yarn test:unit` 和相关 E2E 测试通过。

## 边界情况

- 模块名为空、全空格或过长时，应阻止保存或给出明确校验反馈。
- 字段标题为空时，应允许保存但在编辑区显示安全占位，或阻止保存并提示用户；具体实现需要保持一致。
- 自定义字段数量很多时，弹窗内部应滚动，底部操作按钮仍然容易访问。
- 列表区没有列表项时，预览区应隐藏空列表或显示简洁占位，不应渲染空项目符号。
- 字段宽度配置在窄屏下需要自动降级为单列，避免横向滚动。
- 删除字段后，已填写内容应随字段删除；撤销时应能恢复字段和内容。
- 多个自定义模块可以共存，更新和删除必须按模块 id 和字段 id 处理，不能按模块名匹配。
- 旧版本 localStorage 中没有自定义模块 schema 时，编辑器应正常启动并走默认数据迁移。

## 假设

- 已确认首版“自由控制位置分布”采用响应式网格、字段顺序拖拽和宽度/高度配置，不做任意像素级绝对定位。
- 自定义模块作为现有简历模块数组中的一种模块类型保存，并复用现有模块排序、删除、重命名、预览同步和持久化能力。
- 已确认首版不实现富文本编辑区。
- 列表区首版是简单文本列表或标题加描述列表，不支持多层嵌套列表。
- 自定义模块结构和内容都保存在本地 Pinia/localStorage 数据中。
- 本阶段只定义规格，不修改产品代码。

## 需要澄清

- 无阻塞澄清项；已确认不做富文本，并接受响应式网格配置。
